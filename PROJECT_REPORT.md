# 생각을 키우는 AI 코치 활용 로봇 수업 자료 개발 보고서

## 1. 개발 개요

본 프로젝트는 학생들이 실제 아두이노 및 로봇 키트를 활용하여 로봇 공학의 기초 원리를 학습할 수 있도록 설계한 웹 기반 수업 자료이다. 단순히 예제 코드를 제공하는 방식이 아니라, 학생이 회로를 구성하고, 코드를 작성하고, 시뮬레이션을 통해 결과를 확인하며, AI 코치의 질문을 통해 스스로 문제를 해결해 나가도록 돕는 것을 목표로 한다.

프로젝트명은 **「생각을 키우는 AI 코치 활용 로봇 수업 자료」**이며, 주요 학습 영역은 다음과 같다.

- 아두이노 실습실
- 로봇팔 실습실
- 메카넘 실습실
- 엔지니어링 노트
- 교사용 대시보드
- 관리자 설정

웹앱은 학생용 학습 플랫폼과 교사용 관리 화면을 함께 포함한다. 학생은 각 실습을 진행하면서 AI 질문 도움, 코드 분석, AI 컴파일, 가상 실행, 엔지니어링 노트 작성을 수행할 수 있다. 교사는 학생들의 학습 진행률, 질문 기록, 엔지니어링 노트, PDF 포트폴리오, AI 학습 상태 분석을 확인할 수 있다.

## 2. 개발 목적

본 자료의 핵심 목적은 학생이 로봇 수업에서 단순히 결과물을 따라 만드는 데 그치지 않고, 문제를 관찰하고 원인을 분석하며 해결 과정을 기록하는 공학적 사고를 경험하도록 하는 것이다.

이를 위해 다음과 같은 학습 흐름을 중심으로 설계하였다.

1. 실제 키트를 활용한 회로 및 기구 구성
2. 예제 코드를 참고한 문제 해결형 코드 작성
3. 코드 분석과 AI 컴파일을 통한 오류 탐색
4. 가상 시뮬레이션을 통한 동작 확인
5. 질문형 AI 코치를 통한 사고 확장
6. 엔지니어링 노트를 통한 문제 해결 과정 기록
7. 교사용 대시보드를 통한 학습 과정 피드백

특히 AI 코치는 정답을 직접 제시하는 도구가 아니라, 학생이 스스로 관찰하고 비교하고 실험할 수 있도록 질문을 던지는 역할로 설계하였다.

## 4. 기술 스택

현재 프로젝트는 순수 HTML, CSS, JavaScript 기반으로 구현하였다. React/Vite로 확장할 수 있도록 모듈 단위의 `import/export` 구조를 사용하였다.

사용 기술은 다음과 같다.

- HTML5
- CSS3
- JavaScript ES Modules
- Canvas API
- Three.js
- Firebase Authentication
- Cloud Firestore
- Firebase Hosting
- Firebase Functions
- Firebase Secret Manager

3D 로봇팔 실습에는 Three.js를 사용하였고, 2D 로봇팔과 메카넘 실습에는 Canvas 기반 시뮬레이션을 사용하였다.

## 5. 로그인 및 사용자 구조

학생과 교사는 Google 계정으로 로그인한다. Firebase Authentication을 사용하여 사용자 인증을 처리하고, Firestore의 `profiles` 컬렉션에 사용자 프로필을 저장한다.

프로필 데이터 예시는 다음과 같다.

```json
{
  "uid": "firebase-user-id",
  "school": "테스트고등학교",
  "studentNumber": "10190",
  "name": "테스트1",
  "role": "student",
  "email": "student@example.com"
}
```

교사용 대시보드는 `role` 값이 `teacher`인 사용자만 접근할 수 있도록 구성하였다. 학생이 클라이언트에서 임의로 교사 권한을 바꾸지 못하도록, 실제 권한은 Firestore 프로필의 역할 값을 기준으로 판단한다.

또한 모바일에서 카카오톡, 네이버, 인스타그램 등의 인앱 브라우저로 접속할 경우 Google 로그인이 차단되는 문제를 고려하여, 앱 화면에서 외부 브라우저 사용 안내를 제공하도록 개선하였다.

## 6. 아두이노 실습실

아두이노 실습실은 학생이 실제 아두이노 키트를 가지고 실습할 수 있도록 구성한 모듈이다. 각 lesson은 독립적인 폴더와 데이터 구조를 가지며, 이후 새로운 센서나 부품을 쉽게 추가할 수 있도록 설계하였다.

아두이노 실습실의 단원 구성은 다음과 같다.

- 1단원 LED
- 2단원 서보모터
- 3단원 DC모터
- 4단원 초음파센서
- 5단원 블루투스

좌측 메뉴에는 부품 단위의 큰 단원만 표시하고, 각 단원에 들어가면 본문에서 Lesson을 선택하는 구조로 정리하였다.

각 Lesson은 다음 요소를 포함한다.

- 오늘의 미션
- 학습 목표
- 진행 순서
- 생각해보기
- 필요한 부품
- 회로도
- 코드 편집기
- 참고 코드
- 코드 분석
- AI 컴파일
- 가상 실행
- 질문형 AI 코치
- 학습 진행 저장

중요한 특징은 예제 코드가 곧 정답이 되지 않도록 구성한 점이다. 예를 들어 LED 켜기 수업에서는 참고 코드는 D13 핀을 사용하는 예제로 제공하고, 실제 미션은 D11 핀에 연결한 LED를 켜는 방식으로 제시한다. 이를 통해 학생은 예제 코드를 그대로 복사하는 것이 아니라, 조건의 차이를 이해하고 코드를 수정해야 한다.

## 7. AI 컴파일과 코드 코치

아두이노 실습실에는 교육용 AI 컴파일 기능을 구현하였다. 실제 컴파일러처럼 바이너리를 생성하는 기능은 아니지만, 학생 코드에서 발생할 수 있는 오류를 분석하고 학습 친화적인 피드백을 제공한다.

AI 컴파일에서 확인하는 항목은 다음과 같다.

- 함수명 오타
- `HIGH`, `LOW` 오타
- `OUTPUT`, `INPUT` 오타
- 괄호 누락
- 세미콜론 누락
- 잘못된 핀 번호
- 회로와 코드의 불일치
- 미션 조건과 참고 코드의 차이
- 논리적 오류

AI 코치는 정답 코드를 직접 제공하지 않고, 학생이 관찰과 실험을 통해 오류 원인을 찾도록 질문한다.

예시:

```text
지금 코드에서 LED가 연결된 핀 번호와 digitalWrite()에 들어간 핀 번호가 같은지 먼저 비교해볼까요?
참고 예제와 오늘의 미션에서 달라진 조건은 무엇인가요?
회로와 코드가 서로 다르다면 어떤 현상이 나타날지 예상해볼 수 있을까요?
```

## 8. 로봇팔 실습실

로봇팔 실습실은 2D 로봇팔, 3D 로봇팔, 물건 옮기기 실습으로 구성하였다. 좌측 메뉴에는 세 가지 하위 콘텐츠가 표시되며, 각 콘텐츠 내부에서 Lesson 흐름을 진행한다.

### 8.1 2D 로봇팔

2D 로봇팔은 Canvas 기반으로 구현하였다. 학생은 숄더와 엘보 각도를 조절하면서 로봇팔의 엔드포인트 좌표가 어떻게 변하는지 확인할 수 있다.

주요 기능은 다음과 같다.

- 2축 로봇팔 시뮬레이션
- 숄더 각도 조절
- 엘보 각도 조절
- 5cm 간격 좌표계 표시
- 엔드포인트 좌표 표시
- 목표 좌표와 현재 좌표의 거리 표시
- 도달 불가능한 좌표 안내
- 여러 지점 이동 경로 시뮬레이션
- 장애물 충돌 감지
- 충돌 시 로봇팔 정지 및 경고 표시

엘보 각도는 숄더와 겹쳤을 때 0도, 숄더와 일직선이 되었을 때 180도가 되도록 기준을 잡았고, 시뮬레이션에서는 실제 로봇팔 동작을 이해하기 쉽도록 회전 방향을 조정하였다.

### 8.2 3D 로봇팔

3D 로봇팔은 Three.js를 사용하여 구현하였다. 학생은 3차원 좌표계에서 베이스, 숄더, 엘보 각도를 조절하며 로봇팔의 TIP 좌표를 확인할 수 있다.

주요 기능은 다음과 같다.

- Three.js 기반 3D 시뮬레이션
- 베이스, 숄더, 엘보 관절 조절
- X, Y, Z 좌표계 표시
- TIP 좌표 표시
- 장애물 배치
- 시작점, 중간점, 도착점 경로 이동
- 장애물 충돌 경고
- 코드 편집기와 가상 실행 연동

### 8.3 물건 옮기기

물건 옮기기는 3축 로봇팔과 그리퍼를 활용한 pick & place 실습이다. 학생은 물체를 잡고, 장애물을 넘고, 도착 구역에 내려놓는 과정을 단계별로 학습한다.

학습 과정은 다음과 같이 세분화하였다.

1. 물체 위로 이동하기
2. 물체 위치로 내려가기
3. 그리퍼를 닫아 물체 잡기
4. 장애물을 넘어 도착 구역 위로 이동하기
5. 도착 구역으로 로봇팔 내리기
6. 도착 구역에서 그리퍼를 열어 물체 놓기

그리퍼는 단순히 벌어지는 형태가 아니라, 서보처럼 회전하여 물체를 잡는 느낌이 나도록 디자인하였다. 그리퍼 각도 탐색 범위는 20도에서 80도 사이로 조정하였다.

## 9. 메카넘 실습실

메카넘 실습실은 메카넘휠의 벡터 원리를 이해하고, 이를 코드와 시뮬레이션으로 연결하도록 구성하였다.

좌측 메뉴 구조는 다음과 같다.

- 동작 구현하기
- 자율주행(S자)
- 블루투스 조종

### 9.1 동작 구현하기

동작 구현하기는 여러 Lesson으로 구성된다.

- Lesson 1 직진과 후진
- Lesson 2 좌우 평행 이동
- Lesson 3 대각선 이동
- Lesson 4 제자리 회전

학생은 미리 만들어진 `forward()`나 `backward()` 함수를 단순히 사용하는 것이 아니라, 네 바퀴의 회전 방향을 직접 설정하고, 각 바퀴의 벡터합을 관찰하면서 이동 방향을 이해한다.

시뮬레이터에서는 다음 기능을 제공한다.

- 네 바퀴 방향 선택
- 각 바퀴의 회전 방향 표시
- 메카넘휠 롤러 45도 표현
- 각 바퀴의 벡터 화살표 표시
- 전체 벡터합 표시
- 로봇의 이동 방향 표시
- 제자리 회전 시 회전 방향 표시

### 9.2 자율주행(S자)

자율주행(S자) 실습은 종이컵으로 구성한 코스를 빠져나가는 수업이다. 컵은 실제 수업 상황을 반영하여 80cm 간격을 기준으로 배치하고, 로봇 크기는 약 15cm 크기로 조정하였다.

학생은 다양한 경로를 시뮬레이션해보며 컵에 부딪히지 않고 통과하는 코드를 작성한다. 장애물에 닿으면 시뮬레이터가 멈추고 충돌 경고가 표시되도록 하였다.

### 9.3 블루투스 조종

블루투스 조종은 가상 컨트롤러를 활용해 메카넘휠 자동차를 조종하는 실습이다.

주요 기능은 다음과 같다.

- 전진
- 후진
- 좌우 평행 이동
- 대각선 이동
- 제자리 회전
- 정지
- 로봇 기준 방향 이동
- 배경 이동 방식의 주행 시각화

로봇이 바라보는 방향이 바뀌었을 때도, 전진 버튼을 누르면 경기장 기준이 아니라 로봇 중심 기준의 앞 방향으로 이동하도록 개선하였다.

## 10. 엔지니어링 노트

엔지니어링 노트는 학생이 실습 과정을 기록하고, 문제 해결 과정을 정리하는 공간이다. 화면은 3단 구조로 구성하였다.

왼쪽 패널:

- 내가 작성한 노트 목록
- 새 노트 작성
- 작성 마감
- 마감 취소

가운데 패널:

- 날짜 선택
- 실습 주제 선택
- 오늘의 활동 내용
- 발생한 문제
- 원인 분석
- 해결 방법
- 알게 된 점 및 느낀 점
- 사진 또는 영상 파일명 표시
- 저장
- 삭제

오른쪽 패널:

- AI 노트 코치
- 기본 코칭 문장
- 질문 입력창
- AI 답변 영역
- 추천 키워드
- 비추천 키워드

엔지니어링 노트는 Firestore의 `engineeringNotes` 컬렉션에 저장된다. 학생은 자신의 계정으로 작성한 노트만 불러올 수 있고, 교사는 교사용 대시보드에서 학생별 노트를 확인할 수 있다.

엔지니어링 노트 작성 진행률은 10회 작성을 기준으로 대시보드에 표시된다.

## 11. 교사용 대시보드

교사용 대시보드는 교사가 학생들의 학습 상황을 한눈에 확인할 수 있도록 구성하였다. 기존의 단순 테이블 구조에서 대시보드형 관리 화면으로 확장하였다.

주요 기능은 다음과 같다.

- 전체 학생 수
- 전체 질문 수
- 전체 노트 수
- 엔지니어링 노트 작성 마감 학생 수
- 최근 질문
- 최근 노트
- 학생 현황 테이블
- 질문 기록 테이블
- 노트 기록 테이블
- PDF 보기
- PDF 다운로드
- 선택 학생 PDF 일괄 다운로드
- 학생 상세 보기
- AI 학습 상태 분석

학생 목록에서는 다음 정보를 확인할 수 있다.

- 학교
- 학번
- 이름
- 이메일
- 노트 수
- 질문 수
- 완료 레슨 수
- 마지막 활동 시간
- PDF 보기
- PDF 다운로드
- 상세 보기

완료 레슨 수는 학생이 완료한 lesson의 수를 기준으로 `(완료 수 / 전체 lesson 수)` 형태로 표시되도록 조정하였다.

## 12. PDF 포트폴리오

교사용 대시보드에서는 학생별 학습 기록을 PDF 형태로 확인하거나 다운로드할 수 있다.

PDF에는 선택에 따라 다음 내용을 포함할 수 있다.

- 엔지니어링 노트만
- AI 질문 기록만
- 엔지니어링 노트와 AI 질문 기록 모두

PDF 제목은 **AI Robot Project**로 정리하였고, 학생 인적사항은 본문에 별도로 표시되도록 하였다. 생성일은 날짜만 표시하고 시간은 제외하였다.

다운로드 파일명은 다음 형식을 기준으로 구성하였다.

```text
학번 이름 포트폴리오.pdf
```

또한 PDF 설정 메뉴를 통해 포함 항목과 파일명 형식을 지정할 수 있도록 구조를 마련하였다.

## 13. AI 학습 상태 분석

교사용 대시보드 하단에는 AI 학습 상태 분석 기능을 추가하였다. 이 기능은 학생들이 작성한 엔지니어링 노트와 AI 질문 기록을 바탕으로 학습 상태를 분석한다.

분석 방식은 두 가지이다.

- 종합 분석
- 학생 개별 분석

종합 분석에서는 전체 학생들의 질문 패턴, 노트 작성 경향, 어려움을 겪는 지점, 다음 수업에서 보완할 내용을 분석한다.

학생 개별 분석에서는 특정 학생을 선택하여 다음 항목을 분석한다.

- 강점
- 어려움 신호
- 노트와 질문 기록 근거
- 다음 피드백 질문
- 개별 지도 전략

AI 분석 역시 클라이언트에서 직접 OpenAI API를 호출하지 않고, Firebase Functions를 통해 호출되도록 구성하였다.

## 14. Firebase 데이터 구조

Firestore에는 다음 컬렉션을 사용한다.

### profiles

학생과 교사의 프로필을 저장한다.

```json
{
  "uid": "user-id",
  "school": "학교명",
  "studentNumber": "학번",
  "name": "이름",
  "role": "student",
  "email": "email@example.com"
}
```

### engineeringNotes

학생의 엔지니어링 노트를 저장한다.

```json
{
  "userId": "user-id",
  "school": "학교명",
  "studentNumber": "학번",
  "studentName": "이름",
  "date": "2026-06-07",
  "topic": "메카넘 실습실 - S자 코스",
  "activity": "오늘의 활동 내용",
  "problem": "발생한 문제",
  "cause": "원인 분석",
  "solution": "해결 방법",
  "reflection": "알게 된 점 및 느낀 점",
  "createdAt": "timestamp",
  "updatedAt": "timestamp"
}
```

### aiQuestions

학생의 AI 질문 기록을 저장한다.

```json
{
  "userId": "user-id",
  "school": "학교명",
  "studentNumber": "학번",
  "studentName": "이름",
  "module": "arduino",
  "lessonId": "led-on",
  "lessonTitle": "LED 켜기",
  "question": "LED가 안 켜져요",
  "aiResponse": "AI 응답",
  "createdAt": "timestamp"
}
```

### progress

학생의 학습 진행 상태를 저장한다.

```json
{
  "userId": "user-id",
  "moduleId": "arduino",
  "lessonId": "led-on",
  "status": "completed",
  "missionSuccess": true,
  "updatedAt": "timestamp"
}
```

## 15. OpenAI API 호출 구조

OpenAI API Key는 클라이언트 코드에 포함하지 않는다. 모든 AI 요청은 Firebase Functions를 통해 처리한다.

구조는 다음과 같다.

```text
브라우저
  -> /api/arduino/compile
  -> Firebase Hosting rewrite
  -> Firebase Functions
  -> OpenAI API
```

Functions에서는 Firebase Secret Manager에 저장된 `OPENAI_API_KEY`를 사용한다.

클라이언트는 다음 API 경로만 호출한다.

- `/api/arduino/compile`
- `/api/arduino/coach`
- `/api/coach`
- `/api/teacher/analyze`
- `/api/admin/settings`

이 구조를 통해 API Key가 브라우저에 노출되지 않도록 하였다.

## 16. 관리자 메뉴

관리자 메뉴에서는 AI API 사용 여부를 켜고 끌 수 있다. 관리자는 비밀번호를 입력해야 접근할 수 있으며, 초기 비밀번호는 `1234`이다.

API 사용을 끄면 모든 사용자의 AI 기능 호출이 제한된다. 이 설정은 Firebase Functions에서 확인하므로 클라이언트에서 우회하기 어렵다.

## 17. 배포 구조

현재 배포는 Firebase Hosting과 Firebase Functions를 사용한다.

배포 URL:

```text
https://robot-ai-class.web.app
```

Firebase 배포 항목은 다음과 같다.

- Hosting
- Functions
- Firestore rules

GitHub Actions 자동배포도 구성하였다.

```text
GitHub main/master 브랜치에 push
-> GitHub Actions 실행
-> Firebase Hosting + Functions + Firestore rules 배포
```

GitHub Actions 워크플로 파일:

```text
.github/workflows/firebase-deploy.yml
```

## 18. 교육적 기대 효과

본 수업 자료는 다음과 같은 교육적 효과를 기대할 수 있다.

### 18.1 문제 해결력 향상

학생은 오류가 발생했을 때 정답을 바로 받는 것이 아니라, AI 코치의 질문을 통해 원인을 스스로 탐색한다. 이를 통해 문제 상황을 관찰하고, 가설을 세우고, 실험을 통해 검증하는 과정을 경험한다.

### 18.2 공학적 사고 과정 강화

엔지니어링 노트를 통해 학생은 활동 내용, 발생한 문제, 원인 분석, 해결 방법, 느낀 점을 구조화하여 기록한다. 이는 공학 설계 과정의 핵심인 반복 개선과 기록 습관을 기르는 데 도움이 된다.

### 18.3 실제 키트와 가상 시뮬레이션의 연결

학생은 실제 아두이노 및 로봇 키트를 사용하면서, 웹앱의 가상 시뮬레이션으로 동작 원리를 미리 확인할 수 있다. 특히 로봇팔과 메카넘휠은 물리적 원리를 시각적으로 확인할 수 있어 개념 이해에 효과적이다.

### 18.4 교사의 피드백 효율 향상

교사는 교사용 대시보드에서 학생별 질문, 노트, 진행률, PDF 포트폴리오, AI 분석 결과를 확인할 수 있다. 이를 통해 수업 중 학생 개별 상황을 빠르게 파악하고 맞춤형 피드백을 제공할 수 있다.

## 20. 결론

본 프로젝트는 로봇 수업에서 학생의 사고 과정을 중심에 두기 위해 개발한 웹 기반 학습 플랫폼이다. 아두이노, 로봇팔, 메카넘휠이라는 세 가지 주요 로봇 학습 영역을 바탕으로, 코드 작성, 시뮬레이션, AI 코칭, 엔지니어링 노트, 교사용 대시보드를 하나의 흐름으로 연결하였다.

특히 AI 기능은 정답 제공이 아니라 질문형 코칭을 목표로 하였으며, 학생이 스스로 관찰하고 실험하고 기록하도록 설계하였다. 또한 Firebase 기반 저장 및 배포 구조를 통해 학생은 어느 기기에서든 자신의 학습 기록을 불러올 수 있고, 교사는 학생들의 학습 과정을 실시간에 가깝게 확인할 수 있다.

따라서 본 자료는 단순한 로봇 예제 사이트가 아니라, 실제 수업 현장에서 학생의 문제 해결력과 공학적 사고를 확장하기 위한 AI 기반 로봇 수업 플랫폼으로 활용될 수 있다.
